## MASTER_LOG.md Format

**Columns:**
- `Class#` - Sequential number
- `Date` - YYYY-MM-DD
- `Topic` - Brief topic
- `Assignment` - Link to assignment file
- `Prep` - Not Started / Draft / Done
- `Prep_QC` - QC iterations: `0` / `1` / `2+` / `✓`
- `Audio?` - ✅ if audio prep exists
- `Class?` - ✅ if class attended
- `Transcript?` - ✅ if transcript downloaded
- `Review?` - ✅ if review doc completed
- `Review_QC` - QC iterations
- `Outline?` - ✅ if outline updated
- `Outline_QC` - QC iterations
- `Doctrinal?` - ✅ if IRAC prewrite exists
- `Policy?` - ✅ if policy prewrite exists
- `Signals` - Count of #EXAM_SIGNAL tags
- `Next Action` - What needs to happen next
- `Notes` - QC issues, blockers, context

---

## QC Process

**QC happens by editing documents in place** (no separate QC docs).

**Tracking:**
- Add QC columns to MASTER_LOG: `Prep_QC`, `Review_QC`, `Outline_QC`
- Each QC pass increments counter: `0` → `1` → `2` → `✓`
- QC findings go in Notes column or in doc frontmatter

**Workflow:**
- Generate draft → QC pass 1 → Edit → QC pass 2 (if needed) → Mark `✓`

## PART 1: STRUCTURE CHECK

### Required Sections Present
- [ ] **Black Letter Rule** — Present and clearly stated
- [ ] **Elements/Test** — Numbered list (3-7 elements typically)
- [ ] **Source Authority** — Cases + statutes + Restatement sections
- [ ] **Key Cases** — At least 2-3 primary cases with basic facts
- [ ] **Statutory/Restatement Text** — Full text or clear citation
- [ ] **Policy Rationales** — Explains "why" not just "what"
- [ ] **Common Applications** — Hypo triggers or fact patterns
- [ ] **Variations & Exceptions** — Edge cases or jurisdictional differences
- [ ] **Cross-References** — Links to related doctrines

## PART 2: ACCURACY CHECK

### Rule Statement
- [ ] Rule is ONE clear sentence (not a paragraph)
- [ ] Rule captures the essence of the doctrine
- [ ] Rule is accurate (not oversimplified to the point of error)
- [ ] Rule uses standard legal terminology

**Rule Quality**: [Excellent / Good / Needs Improvement]

---

### Elements/Test
- [ ] Elements are numbered (1, 2, 3...)
- [ ] Each element has a clear definition
- [ ] Element count is reasonable (not 20 tiny elements, not 2 broad ones)
- [ ] Elements are mutually exclusive (no overlap)
- [ ] Elements collectively cover the doctrine (no gaps)

**Elements Quality**: [Excellent / Good / Needs Improvement]

---

### Source Authority
- [ ] Primary case(s) cited correctly
- [ ] Statutes cited with correct section numbers
- [ ] Restatement sections cited accurately
- [ ] Every case listed appears in _REGISTRY_Cases.md
- [ ] Every case has page number reference to Singer

**Source Quality**: [Excellent / Good / Needs Improvement]

---

### Key Cases
- [ ] At least 2-3 cases listed
- [ ] Each case has: Facts + Holding
- [ ] Facts are 2-3 sentences (not too long, not too short)
- [ ] Holding is clear (what did court decide?)
- [ ] Cases are the MOST important for this doctrine (not marginal)

**Cases Quality**: [Excellent / Good / Needs Improvement]

---

### Policy Rationales
- [ ] Explains "why" courts chose this rule
- [ ] Mentions competing policies (efficiency vs. fairness, etc.)
- [ ] Not just "because courts said so"
- [ ] Connects to broader Property law themes

**Policy Quality**: [Excellent / Good / Needs Improvement]

---

### Hypo Applications / Common Applications
- [ ] Lists specific fact patterns that trigger this doctrine
- [ ] Not generic (specific: "competing hunters" not "disputes over property")
- [ ] Includes at least 2-3 common hypo patterns
- [ ] Patterns are realistic (not edge cases only)

**Applications Quality**: [Excellent / Good / Needs Improvement]

- [ ] Links to related doctrines use correct format
- [ ] Links point to sections that actually exist
- [ ] "See also" references are helpful
- [ ] No dead-end links (test 3-5 random links)

# TEMPLATES Directory

This folder contains reusable templates for the Property Law course workflow.

## Available Templates

### 1. `mapping_crosswalk_template.md`
**Purpose**: Create per-class crosswalks linking topics → sources → cases → policy.
**Use when**: Before each class to map assigned readings to doctrine and exam relevance.
**Output location**: `03_MAPPING/`

### 2. `prep_packet_template.md`
**Purpose**: Generate focused 2-4 page prep packets for each class.
**Use when**: Before each class (60-90 min prep window).
**Output location**: `04_PREP/`
**Key features**:
- Doctrine/statute breakdown
- Policy brief with quotable language
- Case cards with hypo variations
- Cold-call prep
- Prediction tracking

### 3. `review_template.md`
**Purpose**: Capture post-class deltas, corrections, and exam signals.
**Use when**: Immediately after each class (20-30 min).
**Output location**: `06_REVIEW/`
**Key features**:
- Delta analysis (predictions vs. reality)
- Outline update blocks
- Exam signal tracking
- Weak points identification
- Metrics update

### 4. `exam_spec_template.md`
**Purpose**: Analyze past exams to understand format, weighting, and professor's style.
**Use when**: After parsing past exams (before semester starts, if possible).
**Output location**: `03_MAPPING/exam_spec.md`
**Key features**:
- Exam format analysis
- Topic weighting
- Favored sources
- Policy framework
- Hypo patterns
- Grading insights

### 5. `outline_insert_template.md`
**Purpose**: Create modular, exam-ready outline sections.
**Use when**: After each class (from review process) or during outline consolidation.
**Output location**: `07_OUTLINE/`
**Key features**:
- Black letter rule
- Element breakdown
- Key cases
- Statutory text
- Policy rationale
- Exam hypos
- Cross-references

### 6. `metrics_template.md`
**Purpose**: Track time, prediction accuracy, weak topics, and spaced repetition.
**Use when**: Weekly (compile from daily tracking).
**Output location**: `08_METRICS/`
**Key features**:
- Time allocation tracking
- Prediction accuracy scoring
- Weak topics log
- Spaced repetition queue
- Exam signal strength
- Efficiency insights

---

## Workflow Integration

### Before Class (60-90 min):
1. Use `mapping_crosswalk_template.md` → Fill in reading crosswalk
2. Use `prep_packet_template.md` → Create focused prep packet

### After Class (20-30 min):
1. Use `review_template.md` → Capture deltas and corrections
2. Use `outline_insert_template.md` → Create outline blocks from review
3. Update `MASTER_LOG.md` with class data

### Weekly:
1. Use `metrics_template.md` → Compile weekly stats
2. Review weak topics and schedule spaced reps

### Once (Before Semester):
1. Use `exam_spec_template.md` → Analyze past exams

---

## Template Customization

These templates are **starting points**. Adjust as needed:
- Add/remove sections based on what works
- Modify length caps if necessary (but keep strict limits!)
- Tune for professor's specific style after first few classes

---

## Agent Hand-Off Notes

When delegating to another agent:
- Point agent to specific template for the task
- Provide class number, topic, and readings
- Specify output location
- Reference course profile parameters from `MASTER_LOG.md`
